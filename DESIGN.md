# 北洋 Java 面试官技术设计

## 1. 项目概述

`agent2026-interview-agent` 是面向天津大学学生就业实习场景的 Java 后端技术面试训练智能体。

项目目标是模拟真实 Java 后端技术一面，让用户完成从面试提问、回答、追问到复盘的完整训练流程。系统不是普通题库，也不是通用聊天机器人，而是围绕 Java 后端面试场景进行任务规划、动态追问、回答评价和复盘生成的智能体应用。

中文产品名暂定为：**北洋 Java 面试官**。

## 2. 参赛规则适配

本项目面向天津大学 AI 智能体大赛技术实现赛道设计。

规则适配点：

- 仓库名使用 `agent2026-` 前缀。
- 学校 GitLab 仓库作为正式提交平台。
- 所有大模型调用使用比赛专属 `tju-llm` API。
- API Key 仅保存在本地环境变量中，不提交到 Git 仓库。
- 项目提供 `README.md`、`DESIGN.md`、演示截图或视频、版权合规承诺书等提交材料。
- 题卡和示例内容原创整理，不直接复制商业题库或培训材料。
- 用户输入项目经历时提示删除敏感个人信息。

## 3. 目标用户

第一版主要服务准备 Java 后端实习、秋招、春招的天津大学学生。

典型使用场景：

- 面试前快速检查 Java 后端核心知识。
- 模拟技术一面的连续追问。
- 根据自己的项目经历练习项目讲解。
- 训练算法题的口述表达和复杂度分析。
- 根据复盘报告安排下一轮训练。

## 4. 核心功能

### 4.1 Java 核心知识面试

系统围绕 Java 后端高频知识进行模拟提问和动态追问。

首发模块：

- Java 基础与集合
- JVM
- Java 并发
- MySQL
- Redis
- Spring / Spring Boot
- 计算机网络
- 操作系统

每道题包含主问题、答案要点、常见错误、追问问题和评分标准。系统会根据用户回答决定继续追问、提示、换题或结束。

### 4.2 项目经历深挖

用户粘贴项目经历或简历片段后，系统识别技术栈、个人职责和项目亮点，并围绕项目进行追问。

追问方向包括：

- 技术选型原因
- 数据库与缓存设计
- 登录鉴权与权限控制
- 性能瓶颈与优化方案
- 用户在项目中的真实职责

### 4.3 算法思维面试

第一版不做在线判题系统，而是做面试口述型算法训练。

系统关注：

- 解题思路
- 时间复杂度和空间复杂度
- 边界情况
- Java 代码或伪代码表达

### 4.4 综合模拟面试

综合模拟面试把项目经历、Java 核心知识和算法思维组合成一场完整技术一面。

基础流程：

1. 项目或自我介绍切入。
2. 项目经历深挖。
3. Java 核心知识追问。
4. 算法思路题。
5. 生成结构化复盘报告。

### 4.5 复盘报告

每次训练结束后生成复盘报告。

报告包含：

- 总分
- 模块分数
- 答得好的点
- 漏掉的关键点
- 错误或危险表达
- 推荐补强模块
- 下一轮训练建议

## 5. 技术架构

确定技术栈：

- 前端：Vue 3 + TypeScript
- 后端：Spring Boot 3.3.6
- Java：17
- 数据库：MySQL
- 大模型：比赛专属 `tju-llm` API

整体调用链：

```text
Vue 前端
  -> Spring Boot 后端
    -> MySQL
    -> tju-llm API
```

前端不直接调用大模型接口，所有模型请求都由 Spring Boot 后端代理完成，避免 API Key 泄露。

工程目录采用 Monorepo：

```text
apps/
  server/   Spring Boot 后端
  web/      Vue 3 前端
```

## 6. 后端模块设计

后端主要模块：

- `InterviewPlanner`：根据模式、模块和难度生成训练计划。
- `QuestionEngine`：负责抽题和生成追问。
- `AnswerEvaluator`：调用大模型评价用户回答。
- `FollowUpDecider`：决定继续追问、提示、换题或结束。
- `ReportGenerator`：生成复盘报告。
- `SessionManager`：管理训练会话状态。
- `LLMClient`：统一封装 `tju-llm` API 调用。

## 7. 数据设计

第一版以最小数据结构为主。

核心数据：

- 题卡：题目、模块、难度、答案要点、追问、评分点。
- 会话：一次训练的模式、模块、开始时间、结束状态。
- 回答：用户回答、AI 评价、得分、追问记录。
- 报告：总分、模块分数、薄弱点、建议。

第一周 MVP 可以先使用 JSON 题卡和简单数据库表，后续再扩展复杂题库管理。

## 8. API 配置

后端通过环境变量读取学校 API 配置：

```env
TJU_LLM_API_URL=https://ai.tju.edu.cn/api/agent2026/gitlab-42-agent2026-interview-agent/chat/completions
TJU_LLM_API_KEY=replace_with_your_api_key
TJU_LLM_MODEL=tju-llm
```

安全要求：

- 不提交 `.env`。
- 不在日志中打印 API Key。
- 不在前端暴露 API Key。
- 处理 401、429、500 等错误。

## 9. 一周 MVP 范围

一周内优先完成：

- Spring Boot 后端骨架
- Vue 3 前端骨架
- MySQL 连接
- 学校 API 调用
- Java 核心知识题卡
- 基础训练流程
- 简单追问逻辑
- 基础复盘报告

一周内暂不做：

- 语音转文字
- 完整项目经历深挖
- 独立算法专项
- 训练历史
- 登录系统
- 复杂 UI 动效

## 10. 成功标准

第一阶段成功标准：

- 前端页面可访问。
- 后端服务可启动。
- 后端能连接 MySQL。
- 后端能调用学校 `tju-llm` API。
- 用户能完成一次 Java 核心知识训练。
- 系统能根据回答进行追问或评价。
- 系统能生成基础复盘报告。
- 仓库中没有 API Key 或敏感配置。
