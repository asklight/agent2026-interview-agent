# 开发路线图

## 总原则

开发时不要一开始追求完整平台，而是先做出一个能演示、能体验、能反复使用的闭环。

最小闭环：

> 选择训练模式 -> AI 提问 -> 用户回答 -> AI 追问 -> 结束生成复盘报告

每周都要做一次总结，明确本周完成了什么、遇到什么问题、下周要做什么。不要只写代码不复盘。

## 阶段 1：设计与骨架

时间：2026-07-08 至 2026-07-14

目标：

- 确定技术栈。
- 初始化仓库。
- 建立前后端基础结构。
- 写好项目设计文档和 README 初稿。
- 完成比赛 API 调用验证。
- 准备第一批 Java、MySQL、Redis、Spring 题卡。

验收标准：

- 项目能启动。
- 技术栈固定为 Spring Boot + Vue 3 + MySQL。
- 后端能成功调用 `tju-llm`。
- 有一个简单页面可以发起测试请求。
- Git 仓库没有泄露 API Key。
- 学校 GitLab 项目已创建，仓库名使用 `agent2026-` 前缀。
- 如使用 GitHub，需要配置为镜像或备份，不能替代学校 GitLab。

## 阶段 2：Java 核心知识面试 MVP

时间：2026-07-15 至 2026-07-21

目标：

- 完成 Java 核心知识面试入口。
- 支持选择模块和难度。
- 系统能提出主问题。
- 用户能提交回答。
- AI 能给出基础评价。

验收标准：

- 用户能完成至少一轮 Java 核心知识问答。
- 至少支持 Java、MySQL、Redis、Spring 四个模块。
- 每个模块至少有 5 张题卡。

## 阶段 3：动态追问与复盘报告

时间：2026-07-22 至 2026-07-28

目标：

- 加入追问逻辑。
- 加入回答评分。
- 加入复盘报告生成。
- 优化模型输出结构，尽量使用 JSON。

验收标准：

- 用户答得浅时，系统能继续追问。
- 用户答完一场训练后，系统能生成结构化复盘报告。
- 报告包含分数、遗漏点和下一轮建议。

## 阶段 4：项目经历深挖

时间：2026-07-29 至 2026-08-04

目标：

- 增加项目经历输入。
- 从项目描述中识别技术栈和用户职责。
- 根据项目内容生成追问。
- 项目深挖结果进入复盘报告。

验收标准：

- 用户粘贴项目描述后，系统能围绕项目技术栈追问。
- 问题不能泛泛而谈，必须和用户项目内容相关。

## 阶段 5：综合模拟面试

时间：2026-08-05 至 2026-08-11

目标：

- 把项目、Java 核心知识和算法思维组合成一场完整模拟面试。
- 支持 10 到 15 分钟的面试流程。
- 支持中途跳过、提示和结束面试。

验收标准：

- 用户可以完成一场完整 Java 技术一面。
- 面试过程包含项目、核心知识和至少一道算法思路题。

## 阶段 6：体验优化与题库扩充

时间：2026-08-12 至 2026-08-25

目标：

- 扩充题卡数量。
- 优化 UI。
- 增加训练历史。
- 优化复盘报告可读性。
- 修复 API 错误处理和限速问题。

验收标准：

- 至少 6 个模块可用。
- 每个主要模块至少 10 张题卡。
- API 异常时有明确提示，不会导致页面崩溃。

## 阶段 7：公开体验准备

时间：2026-08-26 至 2026-09-01

目标：

- 做好演示路径。
- 仓库改为内部可见。
- 准备邀请同学试用。
- 收集第一批反馈。

验收标准：

- 其他同学可以按照 README 启动或访问项目。
- 至少准备一条完整演示流程：项目经历深挖 -> Java 核心知识 -> 复盘报告。

## 阶段 8：反馈迭代

时间：2026-09-02 至 2026-09-15

目标：

- 根据用户反馈优化题目质量、追问和报告。
- 加入算法思维面试增强。
- 视情况加入语音转文字。

验收标准：

- 解决主要体验问题。
- 训练流程稳定。
- 复盘报告更具体。

## 阶段 9：材料整理与最终提交

时间：2026-09-16 至 2026-09-30

目标：

- 完成 `README.md`。
- 完成 `DESIGN.md`。
- 准备演示截图或视频。
- 放入版权合规承诺书。
- 检查 API Key、敏感信息和版权风险。
- 在 2026-09-30 22:00 前完成最终提交。

验收标准：

- 提交材料完整。
- 项目能稳定运行。
- 演示视频能清楚体现智能体能力。
- 没有泄露 API Key。

## 每周应该做什么

每周固定完成四件事：

1. 写本周目标。
2. 开发并提交代码。
3. 周末写周报。
4. 根据周报调整下周计划。

每周周报应该回答：

- 本周完成了什么？
- 哪个功能现在可以演示？
- 遇到了什么问题？
- 哪些范围要砍掉或延后？
- 下周最重要的一个目标是什么？
