# 一周冲刺计划

## 1. 一周目标

一周内完成可演示的最小闭环：

> Vue 页面选择 Java 核心知识训练 -> Spring Boot 后端调用学校 API -> AI 提问 -> 用户回答 -> AI 追问或评价 -> 生成基础复盘报告

一周内不追求完整最终版，只追求能稳定跑通、能展示、能继续扩展。

## 2. 一周内必须完成

必须完成：

- `apps/server` Spring Boot 3.3.6 + Java 17 后端骨架
- `apps/web` Vue 3 + TypeScript 前端骨架
- MySQL 连接
- 学校 `tju-llm` API 调用
- Java 核心知识题卡
- 基础面试流程
- 简单追问逻辑
- 基础复盘报告
- 根目录 `DESIGN.md`
- README 运行说明

## 3. 一周内不做

暂不做：

- 语音转文字
- 完整项目经历深挖
- 独立算法专项
- 训练历史
- 登录系统
- 排行榜
- 复杂图表
- 漂亮动效

这些功能不影响一周 MVP，不允许阻塞主线。

## 4. 每日安排

### Day 1：项目骨架和提交材料

目标：

- 把工程搭起来。
- 把学校提交需要的基础文档补齐。

任务：

- 创建根目录 `DESIGN.md`。
- 初始化 Spring Boot 项目。
- 初始化 Vue 3 + TypeScript 项目。
- 建立前后端目录结构。
- 增加基础 `.env.example` 使用说明。
- 后端增加 `GET /api/health`。
- 前端能显示项目首页。

验收：

- 后端能启动。
- 前端能启动。
- 前端页面能看到项目名。
- `GET /api/health` 可访问。

### Day 2：MySQL 与学校 API

目标：

- 后端能连数据库。
- 后端能调用学校 `tju-llm` API。

任务：

- 创建 MySQL 数据库。
- 配置 Spring Boot 数据源。
- 实现 `LLMClient`。
- 从环境变量读取 API URL、Key、Model。
- 实现 `POST /api/llm/test`。
- 前端增加 API 测试按钮。

验收：

- 后端启动时数据库连接正常。
- API 测试接口能返回学校模型响应。
- 日志不打印 API Key。

### Day 3：题卡系统

目标：

- 有可用的 Java 后端题目基础。

任务：

- 设计题卡 JSON。
- 准备第一批题卡。
- 模块先只做 Java、MySQL、Redis、Spring。
- 每个模块至少 5 张题卡。
- 后端能读取题卡。
- 前端能展示模块选择。

验收：

- 至少 20 张题卡可用。
- 能按模块抽取题目。
- 每张题卡有主问题、要点、追问和评分点。

### Day 4：Java 核心知识训练主流程

目标：

- 用户能完成一轮基础训练。

任务：

- 前端完成训练页面。
- 用户选择模块。
- 后端创建训练会话。
- 系统返回第一道题。
- 用户提交回答。
- 后端调用学校 API 评价回答。
- 前端展示评价结果。

验收：

- 用户能完成至少一道题的提问、回答、评价。
- 前端和后端流程连通。

### Day 5：追问与复盘报告

目标：

- 让系统不像普通问答，而像面试智能体。

任务：

- 增加追问决策。
- 支持继续追问或进入下一题。
- 限制单题最多追问次数。
- 增加训练结束动作。
- 生成基础复盘报告。

验收：

- 用户回答较浅时，系统能追问。
- 用户结束训练后能看到复盘报告。
- 报告包含分数、薄弱点、遗漏点和下一轮建议。

### Day 6：稳定性和演示路径

目标：

- 把一条演示路径打磨顺。

任务：

- 处理 API 401、429、500。
- 增加加载状态和错误提示。
- 整理页面文案。
- 准备一个固定演示流程。
- 更新 README 运行方式。

验收：

- 从打开页面到生成报告，流程能连续跑完。
- API 报错时页面不崩。
- README 能指导启动项目。

### Day 7：提交检查和周报

目标：

- 保证一周成果可提交、可展示、可继续迭代。

任务：

- 跑完整演示流程。
- 检查仓库没有 API Key。
- 更新 `DESIGN.md`。
- 填写周报。
- 截图保存到 `screenshots/`。
- 提交并推送学校 GitLab 和 GitHub。

验收：

- 一条完整 Java 核心知识训练流程可演示。
- 有基础截图。
- 文档更新。
- GitLab 和 GitHub 已同步。

## 5. 一周完成标准

一周后至少要做到：

- 前端页面可打开。
- 后端服务可启动。
- MySQL 连接正常。
- 学校 API 调用正常。
- 用户能选择 Java 核心知识模块。
- 系统能提问。
- 用户能回答。
- 系统能追问或评价。
- 系统能生成基础复盘。
- README 和 DESIGN 可用。
- 仓库没有密钥泄露。

## 6. 一周内的砍范围规则

如果进度落后，按顺序砍：

1. 砍页面美化。
2. 砍训练历史。
3. 砍项目经历深挖。
4. 砍算法专项。
5. 砍语音输入。
6. 砍复杂报告图表。

不能砍：

- 学校 API 调用。
- Java 核心知识训练。
- 追问能力。
- 基础复盘报告。
- README 和 DESIGN。

## 7. 每天提交要求

每天至少一次提交。

建议提交节奏：

- Day 1：`chore: initialize project skeleton`
- Day 2：`backend: add tju llm client`
- Day 3：`feature: add question cards`
- Day 4：`feature: add interview session flow`
- Day 5：`feature: add follow-up and report`
- Day 6：`polish: improve demo flow`
- Day 7：`docs: update submission materials`

每次提交后同步：

```bash
git push
git push github
```
